Ambient Audio

Ambient audio is a type of sound effect that is used to add atmosphere to a game. For example, wind, city noise, rising sound, and more. The AmbientAudio class makes it easy to work with background sound, has fading, etc. 
public class AmbientAudio {
    public AmbientAudio(String);
    public void handleAmbient(float, float, float, float);
    public void handleAmbient(float, float);
    public void handleAmbient();
    public void setLoop(boolean);
    public void play();
    public void stop();
    public Raylib.Music getRlMusic();
    public void unloadAmbient();
}
  • AmbientAudio(String):

    This is the constructor of the AmbientAudio class. It initializes the ambient audio by setting up the audio device and loading the music stream from a specified file path. It ensures the audio device is ready and the music stream is valid.

  • void handleAmbient(float, float, float, float):

    This method manages the ambient audio playback with optional fade-in and fade-out effects. It adjusts the volume of the music based on the playback time and the specified fade parameters.

  • void handleAmbient(float, float):

    This is an overloaded version of the handleAmbient method. It provides a simpler way to manage ambient audio playback with only fade-in and fade-out effects.

  • void handleAmbient():

    This is another overloaded version of the handleAmbient method. It handles ambient audio playback without any fade effects.

  • void setLoop(boolean):

    This method sets whether the ambient audio should loop. If looping is enabled, the music will restart from the beginning once it finishes playing.

  • void play():

    This method starts playing the ambient audio.

  • void stop():

    This method stops the playback of the ambient audio.

  • Raylib.Music getRlMusic():

    This method returns the underlying Raylib.Music object used for managing the audio stream. It allows direct interaction with the music stream if needed.

  • void unloadAmbient():

    This method unloads the ambient audio, freeing up resources associated with the music stream.

Spatial Audio

Spatial audio is a technique for simulating real audio that allows you to convey sound position and loudness. That is, you will hear the sound in the right earphone if the sound is to your right. 
public class SpatialAudio {
    public SpatialAudio(SpatialAudioData);
    public void play();
    public void stop();
    public void handleSpatiality(Vector3Df, float, boolean);
    public float calcDistVol(Vector3Df);
    public float calcDistPan(float, float, boolean);
    public Raylib.Sound getAudio();
    public void setAudioPos(Vector3Df);
    public Vector3Df getAudioPos();
    public float getAudioStartVol();
    public float getAudioCurrVol();
    public float getAudioCurrPan();
    public void setAudioLoudness(float);
    public float getAudioLoudness();
    public void setAudioPSens(float);
    public float getAudioPSens();
    public void unloadSound();
    public static Vector3Df getCamPos(Raylib.Camera3D);
}
  • SpatialAudio(SpatialAudioData):

    This constructor initializes the SpatialAudio object with the given spatial audio data. It ensures the audio device is ready and the sound data is valid.

  • void play():

    This method starts playing the spatial audio.

  • void stop():

    This method stops the playback of the spatial audio.

  • void handleSpatiality(Vector3Df, float, boolean):

    This method adjusts the volume and pan of the audio based on the camera position and orientation. It updates the spatial audio data to reflect these changes.

  • float calcDistVol(Vector3Df):

    This method calculates the volume of the audio based on the distance between the audio source and the camera position. The volume decreases as the distance increases.

  • float calcDistPan(float, float, boolean):

    This method calculates the pan (left-right balance) of the audio based on the camera's position and target orientation. It can apply an inverse effect if specified.

  • Raylib.Sound getAudio():

    This method returns the underlying Raylib.Sound object used for managing the audio stream.

  • void setAudioPos(Vector3Df):

    This method sets the position of the audio source in the 3D world.

  • Vector3Df getAudioPos():

    This method returns the current position of the audio source in the 3D world.

  • float getAudioStartVol():

    This method returns the initial volume of the audio.

  • float getAudioCurrVol():

    This method returns the current volume of the audio, which may have been adjusted based on the distance to the camera.

  • float getAudioCurrPan():

    This method returns the current pan (left-right balance) of the audio, which may have been adjusted based on the camera's position and orientation.

  • void setAudioLoudness(float):

    This method sets the loudness level of the audio. Loudness affects how the volume is perceived at different distances.

  • float getAudioLoudness():

    This method returns the current loudness level of the audio.

  • void setAudioPSens(float):

    This method sets the sensitivity of the audio's pan effect, determining how responsive the pan adjustment is to changes in position.

  • float getAudioPSens():

    This method returns the current sensitivity of the audio's pan effect.

  • void unloadSound():

    This method unloads the sound data, freeing up resources associated with the audio stream.

  • static Vector3Df getCamPos(Raylib.Camera3D):

    This method returns the position of the camera as a Vector3Df object, which can be used for spatial audio calculations.

Quad Tree.

VSDK implementation of QuadTree. What is a quadtree?: A quadtree is a tree data structure in which each internal node has exactly four children. Quadtrees are the two-dimensional analog of octrees and are most often used to partition a two-dimensional space by recursively subdividing it into four quadrants or regions. The data associated with a leaf cell varies by application, but the leaf cell represents a "unit of interesting spatial information". In computer graphics and spatial indexing, quadtrees are incredibly useful for efficiently managing and querying spatial information. They are often used in applications such as image compression, where regions of an image are subdivided to identify areas of uniform color or detail. Quadtrees are also fundamental in collision detection algorithms in gaming and simulations, where the space can be partitioned to quickly locate objects that may intersect or come into contact. The VSDK implementation of QuadTree is designed to handle a fixed maximum number of points before splitting into subnodes, allowing it to efficiently manage spatial data even as the dataset grows. This implementation supports clear operations, dynamic splitting, and efficient retrieval of points, making it a versatile tool for developers working with spatial data. 
public class QuadTree {
    public static final int MAX_LEVELS = 5;
    public static final int MAX_POINTS = 4;

    public List<int[]> points;
    public QuadTree[] nodes;
    public int level;
    public int[] bounds;

    public QuadTree(int, int[]);
    public void clear();
    public void split();
    private int getIndex(int[]);
    public void insert(int[]);
    public List<int[]> retrieve(List<int[]>, int[]);
}
  • public static final int MAX_LEVELS:

    This constant represents the maximal number of recursive subdivisions.

  • public static final int MAX_POINTS:

    This constant represents the maximum number of points a node can hold before it splits.

  • public List<int[]> points:

    This field represents the container for points within the node.

  • public QuadTree[] nodes:

    This field represents the four child nodes of the current node.

  • public int level:

    This field represents the current level of the node within the quadtree.

  • public int[] bounds:

    This field represents the bounding box of the current node.

  • public QuadTree(int, int[]):

    This constructor initializes the quadtree node with a specified level and bounds.

  • public void clear():

    This method clears all points and child nodes in the quadtree.

  • public void split():

    This method splits the current node into four subnodes, dividing the bounding box into four equal quadrants.

  • private int getIndex(int[]):

    This method determines which quadrant of the current node's bounds contains the given point.

  • public void insert(int[]):

    This method inserts a point into the quadtree, splitting the node if it exceeds capacity.

  • public List<int[]> retrieve(List<int[]>, int[]):

    This method retrieves all points that could collide with the given point.
